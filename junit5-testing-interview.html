🧪 JUnit5+Mockito测试面试10题

Java后端实习生面试 · 2026年5月 · 每题含标准答案

Q1：JUnit5和JUnit4有什么区别？

核心变化：JUnit5 = JUnit Platform + Jupiter + Vintage。注解从org.junit.Test变成org.junit.jupiter.api.Test。
新增特性：@DisplayName（可读测试名）、@ParameterizedTest（参数化）、@Nested（嵌套测试）、扩展模型替代Runner。
面试提示：提到JUnit5的@ExtendWith(MockitoExtension.class)替代了JUnit4的@RunWith(MockitoJUnitRunner.class)。

Q2：Mockito中 @Mock 和 @Spy 有什么区别？

@Mock：完全虚拟对象，所有方法默认返回null/0/false，白纸一张。
@Spy：部分模拟，包裹真实对象。不stub的方法走真实逻辑。

@Spy
List spyList = new ArrayList<>();
spyList.add("hello");  // 真实add
Mockito.when(spyList.size()).thenReturn(100); // 只mock size()
// spyList.size() → 100
// spyList.get(0) → "hello"（真实逻辑）
一句话：@Mock全假，@Spy半假半真。@Spy危险在于忘了stub会走真实逻辑。

Q3：Spring Boot测试中 @MockBean 和 @Mock 有什么区别？

@Mock：纯Mockito注解，创建mock但不注入Spring容器。
@MockBean：Spring Boot提供，创建mock并替换Spring容器中的同名Bean。

使用场景：集成测试（@SpringBootTest）中用@MockBean替换外部依赖（如第三方API），单元测试中用@Mock。
注意：@MockBean会导致Spring重新加载ApplicationContext，影响测试性能。

Q4：怎么测试Controller层？

两种方式：
1. 纯Mockito（快）： 
@InjectMocks SeckillController controller;
@Mock SeckillService service;
// 直接调方法，验证返回值
2. MockMvc（真）： 
@WebMvcTest(SeckillController.class)
@AutoConfigureMockMvc
class SeckillControllerTest {
    @Autowired MockMvc mockMvc;
    @MockBean SeckillService service;
    
    @Test void doSeckill_Success() throws Exception {
        when(service.doSeckill(1L,1L))
            .thenReturn(R.ok("秒杀成功"));
        mockMvc.perform(post("/seckill/order")
                .param("stockId","1").param("userId","1"))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.code").value(200));
    }
}

Q5：单元测试覆盖率多少算合格？

行业标准：行覆盖率≥80%、分支覆盖率≥70%。核心业务逻辑（秒杀下单）必须100%覆盖。
工具：JaCoCo插件（Maven/Gradle），mvn test jacoco:report生成HTML报告。
一句话：覆盖率不是目的——发现不了的bug才是。高覆盖率≠高质量测试。

Q6：TDD红-绿-重构是什么？

三步循环：红（写失败测试）→绿（写最少代码让测试通过）→重构（优化代码不破坏测试）。
面试能说的：秒杀项目的Lua脚本就用了TDD——先写测试验证"库存100→扣减1→剩99"，再写Lua脚本，最后重构去重。

Q7：怎么验证方法被调用了？

// 验证service.doSeckill(1L, 1L) 被调用了1次
verify(seckillService, times(1)).doSeckill(1L, 1L);
// 验证从未调用
verify(seckillService, never()).doSeckill(anyLong(), anyLong());
// 至少一次
verify(seckillService, atLeastOnce()).doSeckill(anyLong(), anyLong());
常用验证：times(n) atLeastOnce() never() atMost(n)
