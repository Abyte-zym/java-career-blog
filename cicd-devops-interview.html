1. Git merge 和 Git rebase 的区别？什么时候用哪个？

面试官意图：考察版本控制底线能力——90%实习生说不清楚
merge：保留完整历史，产生合并提交。适合公共分支（main）、团队协作。
rebase：变基到目标分支顶端，历史线性干净。适合个人分支整理，禁止对公共分支rebase。
一句话：merge保留真相，rebase改写历史。团队协作优先merge，个人分支用rebase整理后merge。

2. Git reset 和 Git revert 区别？

面试官意图：考察你对Git底层操作的理解
reset：移动HEAD指针到指定commit，改写历史。分三档：--soft(保留暂存和工作区)、--mixed(保留工作区)、--hard(全部丢弃)。
revert：生成一个新的反向commit来撤销，不改写历史。
关键：reset用于本地整理，revert用于远程分支（安全）。已推送的commit用revert。

3. CI/CD 完整流程是什么？你项目中怎么做的？

面试官意图：考察DevOps认知——知道代码怎么从本地到线上
CI（持续集成）：代码push → 自动构建 → 自动测试 → 代码扫描 → 反馈结果
CD（持续交付/部署）：CI通过 → 构建镜像 → 推送仓库 → 部署到环境 → 冒烟测试
我的实践：秒杀项目用docker-compose一键部署，RAG项目加了Actuator监控 + 优雅停机。

4. Docker 镜像分层机制？如何减小镜像体积？

面试官意图：考察Docker底层原理
分层原理：每个Dockerfile指令创建一个只读层，最终叠加成容器。相同层可复用（缓存加速）。
优化手段： 1. 多阶段构建（Maven打包+运行时分开，最终镜像不含JDK） 2. 合并RUN指令减少层数 3. 用.dockerignore排除无用文件 4. 选择alpine/slim基础镜像

5. GitHub Actions 核心概念？

面试官意图：考察CI工具实操经验
核心三要素：
- workflow：定义在.github/workflows/*.yml，由事件触发
- job：一个workflow包含多个job，可并行/串行
- step：每个job包含多个step（run脚本或用actions）
常见场景：push自动跑测试、PR自动代码检查、tag自动构建镜像。

6. Kubernetes 核心概念（Pod/Deployment/Service）

面试官意图：考察K8s基础认知
Pod：最小调度单元，包含1个或多个容器，共享网络/存储。
Deployment：声明式管理Pod副本数、滚动更新策略、回滚。
Service：为Pod提供稳定的访问入口（ClusterIP/NodePort/LoadBalancer），通过Label Selector匹配。
关系：Deployment管理Pod → Service暴露Pod。

7. 蓝绿部署 vs 滚动发布 vs 灰度发布？

面试官意图：考察生产部署策略
蓝绿：两套完整环境，切换流量。优点：秒级切换、零停机。缺点：资源翻倍。
滚动：逐个替换实例，始终有实例在服务。K8s默认策略。
灰度（金丝雀）：先放1%流量验证新版本，逐步扩大。最安全但最复杂。
选择：小团队用滚动，大流量用灰度，金融系统用蓝绿。

8. 什么是 IaC（基础设施即代码）？

面试官意图：考察运维自动化思维
定义：用代码（而非手动操作）管理和配置基础设施。
工具：Terraform（多云）、Ansible（配置管理）、Docker Compose（本地编排）、K8s YAML。
优势：版本控制、可重复、自动化、团队协作。
我的实践：项目用docker-compose.yml定义所有服务，git clone后一条命令拉起。

9. 日志收集和监控怎么做？

面试官意图：考察可观测性认知
日志：应用输出到stdout → Docker采集 → ELK/Loki存储 → Grafana/Kibana展示
监控：Prometheus采集指标 → Grafana可视化 → AlertManager告警
链路追踪：traceId贯穿所有服务 → Jaeger/SkyWalking查看调用链
我的实践：项目加了MDC traceId + Actuator + @Slf4j结构化日志。

10. 怎么保证线上服务不被打垮？

面试官意图：考察稳定性保障全貌
事前：压测（JMeter）、限流（Sentinel）、资源隔离（线程池/连接池）
事中：熔断降级（Sentinel/CircuitBreaker）、弹性扩容（K8s HPA）
事后：日志回溯、告警复盘、故障演练
我的实践：秒杀项目用Sentinel做QPS限流+熔断降级，RocketMQ削峰保护下游。
