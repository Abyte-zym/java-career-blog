1. 什么是窗口函数？ROW_NUMBER/RANK/DENSE_RANK 区别？

答案：窗口函数在不改变行数的情况下进行聚合计算。ROW_NUMBER不处理并列，RANK并列跳号，DENSE_RANK并列不跳号。

SELECT name, score,
  ROW_NUMBER() OVER(ORDER BY score DESC) as rn,
  RANK() OVER(ORDER BY score DESC) as rk,
  DENSE_RANK() OVER(ORDER BY score DESC) as dr
FROM student;

追问：PARTITION BY和ORDER BY同时用时，窗口函数怎么执行？先PARTITION分组，再在每个组内按ORDER BY排序计算。

2. CTE（公用表表达式）是什么？递归CTE怎么用？

答案：CTE是临时命名的结果集，可被后续查询引用。递归CTE用于处理树形结构（组织架构、评论回复）。

WITH RECURSIVE org_tree AS (
  SELECT id, name, parent_id, 1 as level FROM org WHERE parent_id IS NULL
  UNION ALL
  SELECT o.id, o.name, o.parent_id, t.level+1
  FROM org o JOIN org_tree t ON o.parent_id = t.id
) SELECT * FROM org_tree;

6. 分库分表怎么设计？水平拆分 vs 垂直拆分？

答案：垂直拆分按业务模块（用户库、订单库），水平拆分按数据行（单表过大时按user_id取模分片）。常用中间件：ShardingSphere。核心问题是跨库JOIN和分布式事务——解法是冗余字段+最终一致性。

7. MySQL事务隔离级别及各自解决的问题？

READ UNCOMMITTED：无锁，有脏读。READ COMMITTED：解决脏读，有不可重复读。REPEATABLE READ（MySQL默认）：解决不可重复读，有幻读（InnoDB用间隙锁解决）。SERIALIZABLE：全部解决，性能最差。

8. InnoDB的MVCC怎么实现的？

答案：基于undo log版本链+ReadView。每行有隐藏列trx_id和roll_pointer。事务开始时生成ReadView，通过判断trx_id是否在活跃事务列表中决定可见性。READ COMMITTED每次查询都生成新ReadView，REPEATABLE READ第一次查询生成。

9. SQL联合查询：INNER JOIN/LEFT JOIN/RIGHT JOIN/FULL JOIN 区别及应用场景？

INNER JOIN取交集，LEFT JOIN以左表为准右表补NULL，RIGHT JOIN相反，FULL JOIN（MySQL不支持，可用UNION实现）取并集。实战中LEFT JOIN最常见——查用户及其订单"SELECT u.*, o.amount FROM user u LEFT JOIN orders o ON u.id=o.uid"。

10. 如何设计一个高并发的数据库架构？

答案：读写分离（主写从读）、分库分表（降低单库压力）、缓存层（Redis挡热点数据）、连接池调优（HikariCP）、SQL预编译（防注入+提升性能）、监控告警（慢查询+P99延迟）。
