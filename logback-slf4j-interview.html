Q1：SLF4J和Logback是什么关系？为什么用SLF4J门面？

门面模式（Facade Pattern）：SLF4J是日志门面（接口），Logback/Log4j2是日志实现（引擎）。你的代码只依赖SLF4J接口，换日志框架不改代码。

类比：SLF4J=USB接口标准，Logback=具体的USB设备。换设备不用改接口。

// 代码里永远只写SLF4J
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
private static final Logger log = LoggerFactory.getLogger(MyClass.class);
log.info("用户登录成功, userId={}", userId);

Q2：日志级别有哪些？生产环境用什么级别？

5个级别（从低到高）：
TRACE → 最细粒度，追踪每步（开发调试用）
DEBUG → 调试信息（开发环境）
INFO → 关键业务流程（生产环境默认）
WARN → 潜在问题（如接近限流阈值）
ERROR → 错误异常（必须记录堆栈）

面试金句：生产环境用INFO级别，关键业务节点打INFO（如"订单创建成功"），异常打ERROR并附带完整堆栈。DEBUG只在排查问题时临时开启。

Q3：logback-spring.xml里怎么区分dev和prod环境？

用 <springProfile> 标签： 
<configuration>
    <!-- dev环境：控制台输出，DEBUG级别 -->
    <springProfile name="dev">
        <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
            <encoder>
                <pattern>%d{HH:mm:ss} [%thread] %-5level %logger{36} - %msg%n</pattern>
            </encoder>
        </appender>
        <root level="DEBUG">
            <appender-ref ref="CONSOLE"/>
        </root>
    </springProfile>

    <!-- prod环境：文件输出，INFO级别 -->
    <springProfile name="prod">
        <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
            <file>/var/log/app.log</file>
            <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
                <fileNamePattern>/var/log/app.%d{yyyy-MM-dd}.log</fileNamePattern>
                <maxHistory>30</maxHistory>
            </rollingPolicy>
        </appender>
        <root level="INFO">
            <appender-ref ref="FILE"/>
        </root>
    </springProfile>
</configuration>

Q4：MDC是什么？怎么实现分布式链路追踪？

MDC（Mapped Diagnostic Context）：线程级别的日志上下文Map。把traceId放进去后，该线程所有日志自动带traceId。

使用： 
// 在Filter或Interceptor中设置traceId
MDC.put("traceId", UUID.randomUUID().toString().replace("-",""));
// 日志pattern中引用：%X{traceId}
// 输出：[2026-05-22 14:30:00] [http-nio-8080-1] INFO  c.z.service.OrderService - [abc123] 订单创建成功
日志配置：<pattern>%d{yyyy-MM-dd HH:mm:ss} [%thread] %-5level %logger{36} - [%X{traceId}] %msg%n</pattern>

面试加分：跨服务用Feign拦截器把traceId放到HTTP Header里，下游服务从Header取出来设MDC。ELK里用traceId聚合所有服务的日志。

Q5：AsyncAppender是什么？什么时候用？

异步日志追加器：把日志写入操作放到独立线程，不阻塞业务线程。

适用场景：高并发下日志量大、写磁盘成为瓶颈时。
<appender name="ASYNC" class="ch.qos.logback.classic.AsyncAppender">
    <queueSize>512</queueSize>
    <discardingThreshold>0</discardingThreshold>
    <neverBlock>true</neverBlock>
    <appender-ref ref="FILE"/>
</appender>
关键参数：queueSize=512 缓冲区大小、neverBlock=true 队列满时丢弃日志而非阻塞业务。

一句话：异步不阻塞业务，但丢日志风险高。金融交易场景慎用。

Q6：日志滚动策略怎么配？

两种滚动：
1. 按时间滚动：每天/每小时一个新文件
2. 按大小滚动：文件超过100MB切新文件
<rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
    <fileNamePattern>/var/log/app.%d{yyyy-MM-dd}.%i.log</fileNamePattern>
    <maxFileSize>100MB</maxFileSize>
    <maxHistory>30</maxHistory>
    <totalSizeCap>3GB</totalSizeCap>
</rollingPolicy>
最佳实践：时间+大小双重滚动、保留30天、总容量上限3GB。防止磁盘写爆。

Q7：Logger的继承关系是什么？root logger和子logger的关系？

Logger是树形继承结构：com.zhangyanming.rag.controller.ChatController 的logger会继承 com.zhangyanming.rag 的配置，最终继承root。

<!-- 全局默认INFO -->
<root level="INFO">
    <appender-ref ref="FILE"/>
</root>

<!-- 自己的包开DEBUG -->
<logger name="com.zhangyanming" level="DEBUG" additivity="false">
    <appender-ref ref="FILE"/>
</logger>

<!-- 第三方框架关掉噪音 -->
<logger name="org.springframework" level="WARN"/>
additivity="false"：不向父logger传递，避免重复输出。

Q8：生产环境遇到ERROR怎么快速定位到代码位置？

三步定位法：
1. 看日志格式包含位置信息：%logger{36}:%L → 输出类名:行号
2. ERROR必须打堆栈：log.error("订单处理失败, orderId={}", orderId, exception); 第三个参数传异常
3. ELK里traceId聚合：同一个traceId串联整个请求链路的所有ERROR

面试加分：提一嘴 Arthas——生产环境不改代码不重启就能看方法入参和返回值。

Q9：Spring Boot默认的日志配置是什么？怎么自定义？

Spring Boot默认：Logback + SLF4J，配置文件 logback-spring.xml 放resources下自动加载。

application.yml里也能配： 
logging:
  level:
    com.zhangyanming: DEBUG      # 自己的包DEBUG
    org.springframework: WARN    # Spring框架WARN
  file:
    path: /var/log                # 日志目录
    name: /var/log/app.log        # 日志文件名
application.yml vs logback-spring.xml：简单配置用yml，复杂配置（异步/滚动/MDC）用xml。xml优先级更高。

Q10：日志打印有什么规范？哪些东西不该打日志？

必须打日志：关键业务节点（下单/支付/登录）、外部API调用（请求+响应）、异常（ERROR级别+堆栈）、定时任务开始/结束。

绝对不打：密码/Token/身份证号/手机号（脱敏后打掩码）、超大对象（toString几KB）、循环里打INFO。

参数化避免字符串拼接： 
// ❌ 字符串拼接，即使DEBUG不输出也执行了拼接
log.debug("用户" + user.getName() + "登录, IP=" + ip);
// ✅ 参数化，DEBUG不输出时参数方法不执行
log.debug("用户{}登录, IP={}", user.getName(), ip);
