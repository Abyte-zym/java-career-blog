Q1：Tomcat默认最大线程数是多少？怎么调？

默认：maxThreads=200，minSpareThreads=10。

调优公式：
<Connector port="8080" protocol="HTTP/1.1"
    maxThreads="500"        
    minSpareThreads="25"    
    acceptCount="200"       
    connectionTimeout="20000"
/>
面试加分：提一句不要盲目调到1000——线程过多导致频繁上下文切换，反而吞吐量下降。压测后看CPU利用率来决定。

Q2：Tomcat的BIO、NIO、APR三种模式有什么区别？

BIO（Blocking IO）：一个连接一个线程，连接不干活的线程也阻塞等。Tomcat7之前默认。并发差。

NIO（Non-blocking IO）：一个线程处理多个连接，Selector轮询就绪的Channel。Tomcat8+默认。适合高并发短连接。

APR（Apache Portable Runtime）：JNI调用操作系统级IO，性能最高。需要额外装native库。适合静态资源和高并发长连接。

一句话总结：NIO对付日常高并发够用，APR追求极致性能时启用。Spring Boot默认就是NIO。

Q3：Tomcat怎么优化JVM参数？

关键JVM参数（catalina.sh 或 setenv.sh）： 
JAVA_OPTS="-Xms2048m -Xmx2048m      # 堆内存初始=最大
-XX:+UseG1GC                        # G1垃圾回收器
-XX:MaxGCPauseMillis=200            # GC最大停顿200ms
-XX:+PrintGCDetails                 # 打印GC日志
-Djava.security.egd=file:/dev/./urandom"  # 加速随机数生成
面试要点：Spring Boot内嵌Tomcat改JVM参数在启动命令里加-Xmx。生产环境堆内存设物理内存的50-70%。

Q4：Tomcat的类加载机制和JVM的有什么区别？

Tomcat打破了双亲委派：
- WebappClassLoader：优先加载自己WEB-INF/classes，再委托给父加载器
- 目的：不同Web应用使用同一库的不同版本互不干扰
- 加载顺序：Bootstrap → System → Common → Webapp（反过来先找自己的）

实战意义：两个应用一个用Spring 5一个用Spring 6放同一Tomcat不会冲突。

Q5：Tomcat的Session共享怎么实现？

三种方案：
1. Redis集中存储：Spring Session + Redis。所有Tomcat读写同一Redis。
2. Tomcat Cluster：组播复制Session，节点间广播。大集群性能差。
3. IP哈希负载均衡：Nginx ip_hash让同一用户永远打到同一台Tomcat。

面试推荐回答：生产环境用Spring Session + Redis——简单可靠，不侵入代码。

Q6：Tomcat和Jetty有什么区别？什么场景用哪个？

核心差异：
| Tomcat | Jetty | |--------|-------| | 体积大，功能全 | 体积小，嵌入式首选 | | Servlet容器完整 | 更像一个Servlet库 | | Spring Boot默认(到2.x) | 可嵌入单元测试启动 | | 社区更大 | 更灵活可定制 | | 稳定性优先 | 轻量化优先 |
选型建议：传统Web项目用Tomcat。需要嵌入式（如微服务内嵌容器、单元测试启动）用Jetty。Spring Boot可以用Jetty替代Tomcat——改个pom依赖就行。

Q7：Tomcat的Connector和Container是什么关系？

Tomcat架构 = Connector + Container：
- Connector（连接器）：负责网络通信——接收HTTP请求，解析成ServletRequest
- Container（容器）：负责处理请求——Engine→Host→Context→Wrapper（责任链模式）
- 关系：Connector把请求交给Engine，Engine路由到对应Host→Context→Wrapper（Servlet）

面试金句："Connector管网络IO，Container管Servlet生命周期。两者解耦是Tomcat架构的核心。"

Q8：Tomcat内存溢出（OOM）怎么排查？

四步排查法：
1. 看日志：catalina.out 有 OutOfMemoryError 和堆栈
2. 看是哪种OOM：Java heap space→增大-Xmx；PermGen/Metaspace→增大-XX:MaxMetaspaceSize
3. 生成dump：-XX:+HeapDumpOnOutOfMemoryError→MAT/JProfiler分析
4. 常见原因：Session不超时导致堆积/静态集合一直add不remove/线程池ThreadLocal没清理

面试加分：提到Arthas的memory命令可以实时监控内存使用。

Q9：Spring Boot内嵌Tomcat怎么调参数？

两种方式：
方式一（application.yml）： 
server:
  tomcat:
    threads:
      max: 200
      min-spare: 20
    max-connections: 10000
    accept-count: 200
  connection-timeout: 30s
方式二（代码自定义）： 
@Bean
public TomcatServletWebServerFactory factory() {
    TomcatServletWebServerFactory f = new TomcatServletWebServerFactory();
    f.addConnectorCustomizers(c -> {
        Http11NioProtocol p = (Http11NioProtocol) c.getProtocolHandler();
        p.setMaxThreads(500);
    });
    return f;
}

Q10：Tomcat的war包和jar包部署有什么区别？Spring Boot为什么用jar？

WAR（Web Archive）：部署到外部Tomcat的webapps目录。Tomcat解压→加载。需要外部Tomcat环境。

JAR（Java Archive）：Spring Boot Fat Jar内嵌Tomcat。一个jar包 = Tomcat + 你的代码。直接java -jar启动。

Spring Boot为什么选jar："一次构建到处运行"——不需要预装Tomcat，一个jar包扔服务器就能跑。微服务时代的标准部署方式。

面试时加一句：Fat Jar的BOOT-INF/lib目录下能看到内嵌的Tomcat依赖。
