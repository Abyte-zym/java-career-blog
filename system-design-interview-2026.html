2026 Java后端面试 · 系统设计高频考点 · 每道题含架构思路

Q1：设计一个短URL系统（如TinyURL）

核心考点：哈希算法选择、Base62编码、重定向策略

用户 → API Gateway → 短URL服务 → Redis缓存 → MySQL ↓ 生成算法 longUrl → MD5(前8位) → Base62 → 短码 → 存库 → 返回

关键点：用MurmurHash比MD5碰撞率低；短码设唯一索引防重复；302临时重定向保留统计；预生成批量短码提升并发。

Q2：秒杀系统架构设计

核心考点：分层过滤、库存原子扣减、削峰填谷

CDN静态化 → Nginx限流(令牌桶) → Gateway鉴权 → Redis Lua原子扣减 → RocketMQ异步下单 → MySQL落库 → Sentinel熔断保护 → 前端排队提示

库存用Redis Lua脚本原子操作；消息队列削峰时注意消息表+定时对账保证一致性；热点商品库存分段存储分散压力。

Q3：设计一个Feed流系统（如微博/Twitter）

核心考点：推拉模式选择、时间线生成、缓存策略

粉丝少(< 1万)：推模式 → 发帖时写入所有粉丝的Timeline Redis List 粉丝多(> 1万)：拉模式 → 读时聚合关注者的最近帖子 混合：大V推活跃粉丝 + 普通用户拉

Timeline用Redis Sorted Set（score=时间戳）；热点Feed预计算缓存；分页用ZREVRANGE。

Q4：分布式唯一ID生成方案

核心考点：雪花算法、号段模式、时钟回拨处理

Snowflake (64bit): [1bit未用] [41bit毫秒] [10bit机器ID] [12bit序列号] → 理论409.6万/s，全局递增

美团Leaf方案：号段模式批量取号，减少DB访问；时钟回拨用等待+借用未来时间处理；UUID无序不推荐做数据库主键。

Q5：如何设计一个API限流系统？

核心考点：固定窗口 vs 滑动窗口 vs 令牌桶 vs 漏桶

方案对比： 令牌桶(Sentinel/Guava) → 允许突发，最常用 滑动窗口(Redis ZSet) → 精确但占内存 漏桶 → 恒定速率，不常用

生产方案：Sentinel控制台配置流控规则 + Nacos动态推送；集群限流用Redis+Lua原子计数，单机限流用RateLimiter。

Q6：设计一个实时排行榜（Top 100）

核心考点：Redis Sorted Set、分段排序、定时刷新

Redis ZADD leaderboard score userId ZREVRANGE leaderboard 0 99 → Top100 ZINCRBY leaderboard 1 userId → 更新分数

千万级数据：分段排序——前1000名实时更新，后面的定时批量计算；热度衰减用时间加权分数（score = 原始分 × e^(-λt)）。

Q7：设计"附近的人"功能

核心考点：GeoHash编码、Redis GEO、空间索引

经纬度 → GeoHash编码 → Redis GEOADD locations lng lat userId → GEORADIUS locations lng lat 5 km → 排序返回

GeoHash长度越长精度越高（7位≈153m）；热门区域用GeoHash前缀分组减少扫描；更新频率控制在分钟级避免写放大。

Q8：设计消息推送系统（App Push）

核心考点：长连接管理、离线消息、全量推送

用户设备 → WebSocket长连接 → 推送网关 → Kafka消息队列 → 消费者按userID分发 → 厂商通道(APNs/FCM/华为) → 手机通知栏

连接管理：Netty维护长连接，心跳检测30s；离线消息存Redis等用户上线再推；全量推送走厂商批量接口，限制QPS。

Q9：设计大文件上传系统

核心考点：分片上传、断点续传、秒传检测

前端 → 计算MD5 → 请求初始化(秒传检测) → 分片上传(5MB/片) → Redis记录进度 → 全部完成 → 合并分片 → 存储OSS → 返回URL

秒传：MD5查Redis，相同文件直接返回已有URL；并发上传用Redis Bitmap标记分片状态；合并时校验MD5完整性。

Q10：设计API网关

核心考点：路由转发、鉴权、限流、协议转换

客户端 → API Gateway → 认证(JWT解析) → 限流(令牌桶) → 路由(Nacos服务发现) → 负载均衡 → 后端微服务 → 响应聚合返回

核心组件：Spring Cloud Gateway + Nacos注册中心；自定义GlobalFilter实现鉴权和日志；Resilience4j断路器保护下游。
