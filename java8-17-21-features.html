1. Lambda表达式是什么？和匿名内部类有什么区别？

答： Lambda是Java 8引入的函数式编程语法，本质是函数式接口的简洁实现。和匿名内部类的关键区别：

  • 字节码层面： Lambda不生成独立的.class文件（使用invokedynamic指令），匿名内部类生成Outer$1.class
  • this引用： Lambda中this指向外部类，匿名内部类中this指向内部类自身
  • 变量捕获： Lambda只能引用effectively final的外部变量
// Lambda写法
Runnable r = () -> System.out.println("Hello");

// 匿名内部类写法
Runnable r2 = new Runnable() {
    public void run() { System.out.println("Hello"); }
};

2. Stream API的核心操作有哪些？map和flatMap的区别？

答： Stream操作分为中间操作（惰性）和终止操作（触发执行）：

  • 中间操作：filter、map、flatMap、sorted、distinct、limit、skip、peek
  • 终止操作：collect、forEach、reduce、count、anyMatch、findFirst

map vs flatMap： map是1对1映射，flatMap是1对多映射并将多个流展平为一个流。

// map: 每个元素变一个
List<String> names = users.stream().map(User::getName).toList();

// flatMap: 展平嵌套集合
List<String> allTags = articles.stream()
    .flatMap(a -> a.getTags().stream()).toList();

3. Optional解决了什么问题？怎么正确使用？

答： Optional是Java 8引入的容器类，解决NPE（空指针）问题，强制开发者显式处理null。正确姿势：

  • ✅ orElseThrow() - 为空抛异常
  • ✅ orElseGet(() -> ...) - 惰性提供默认值
  • ✅ map()/flatMap() - 链式转换
  • ❌ 不要用 Optional.get() 直接取值（可能NoSuchElementException）
  • ❌ 不要用Optional作为字段类型或方法参数
String result = Optional.ofNullable(user)
    .map(User::getAddress)
    .map(Address::getCity)
    .orElse("未知城市");

4. Java 17的Records是什么？和Lombok @Data有什么区别？

答： Record是Java 17正式GA的特性，是一种不可变的透明数据载体。编译后自动生成：构造器、equals/hashCode、toString、getter（无前缀get）。

Record vs Lombok：

  • Record是语言级实现，Lombok是编译期注解处理器
  • Record强制不可变（所有字段final），Lombok @Data生成的是可变bean
  • Record不能继承其他类（已继承java.lang.Record），Lombok无此限制
// 一行代码 = 构造器 + getter + equals + hashCode + toString
public record User(Long id, String name, String email) {}

5. Sealed Classes（密封类）解决了什么问题？

答： Java 17引入Sealed Classes，用于限制哪些类可以继承/实现当前类或接口。配合模式匹配实现穷举检查——编译器能验证所有子类型被覆盖。

public sealed interface Result permits Success, Failure {
    record Success(Object data) implements Result {}
    record Failure(String error) implements Result {}
}

// 编译器确保所有分支都被处理
String msg = switch (result) {
    case Result.Success s -> "成功: " + s.data();
    case Result.Failure f -> "失败: " + f.error();
}; // 无需default——编译器已验证穷举

6. Pattern Matching（模式匹配）在Java 17/21中怎么用？

答： 模式匹配让类型检查+类型转换一步完成：

  • Java 17： instanceof模式匹配
  • Java 21： switch模式匹配 + Record解构
// Java 17: instanceof模式匹配
if (obj instanceof String s && s.length() > 5) {
    System.out.println(s.toUpperCase()); // s已自动转为String
}

// Java 21: switch模式匹配
return switch (shape) {
    case Circle(var r) -> Math.PI * r * r;
    case Rectangle(var w, var h) -> w * h;
};

7. Virtual Threads（虚拟线程）是什么？和平台线程的区别？

答： Java 21正式GA的Virtual Threads是轻量级用户态线程，由JVM调度而非操作系统：

  • 平台线程： 1:1映射OS线程，栈~1MB，创建成本高，限于~几千个
  • 虚拟线程： M:N映射到少量OS线程，栈按需分配（堆上），可创建百万级
  • 使用场景： IO密集型任务（一个请求一个虚拟线程），无需线程池
// 创建10万个虚拟线程——以前不可能
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 100_000; i++) {
        executor.submit(() -> {
            Thread.sleep(100); // 阻塞时自动yield，不占用OS线程
            return "done";
        });
    }
}

9. Text Blocks（文本块）和 String 新方法有哪些？

答： Java 15正式GA的Text Blocks用"""包裹多行字符串，自动处理缩进：

String json = """
    {
        "name": "%s",
        "age": %d
    }
    """.formatted("张三", 22);

// Java 11新增的String方法
"  hello  ".strip();        // 去除首尾空白（Unicode-aware）
"".isBlank();               // 判断是否为空或仅空白
"a\nb\nc".lines().count();  // = 3, 按行分割返回Stream
"hello".repeat(3);          // "hellohellohello"

10. var关键字和类型推断有什么限制？

答： Java 10引入的var用于局部变量类型推断，编译器根据初始化表达式推断类型：

  • ✅ 局部变量、增强for循环、try-with-resources
  • ❌ 方法参数、返回类型、字段声明、Lambda参数（Java 11+允许Lambda参数用var）
  • ❌ 未初始化的变量声明（var x; // 编译错误）
  • ❌ 初始化为null（编译器无法推断类型）
var list = new ArrayList<String>();  // 类型推断为ArrayList<String>
var map = Map.of("key", "value");    // Map<String, String>
for (var user : users) { ... }       // 增强for循环中可用
